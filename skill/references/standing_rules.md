# Standing CV Rules (non-negotiable)

Apply on **every** tailored CV. These encode ATS-safety + credibility for biotech/life-science roles. Customize the bracketed ones to your own truth, but keep the structural rules.

## Credibility & honesty
1. **Consistent credential framing.** Write your top credential the same way every time (e.g., a PhD lab, institution, and advisor named identically across versions). Pick the strongest accurate phrasing and never vary it.
2. **Degree status is exact.** "PhD (2022–2026)" if completed; "PhD candidate, expected 202X" if not. Never blur "in progress" and "completed."
3. **No false language proficiency.** List only languages you can actually work in, with an honest level. Don't pad.
4. **No tool/method overclaiming.** If you've read about a technique but not run it in production, say "conceptually fluent / working knowledge," never "hands-on / expert." One overclaim a scientist-reviewer catches sinks the whole CV.
5. **Quantify only what's real.** Prefer honest scale ("team of 5", "6 years", ">6,000× selectivity") over vague superlatives. Never invent numbers.

## ATS-safety (so a machine can read it)
6. **Single column. No tables, text boxes, multi-column layouts, images, icons, or logos.** These scramble ATS parsers — your experience silently disappears.
7. **No content in headers/footers.** Many ATS ignore them.
8. **Standard section headings only:** *Professional Summary · Core Skills · Experience · Education* (+ optional *Certifications / Publications*). Don't get creative with names — the parser keys off them.
9. **Reverse-chronological experience.** Dates on every role as `Company | YYYY – YYYY`.
10. **Mirror the posted job title** in your headline and (truthfully) in the most relevant role.
11. **Keywords verbatim.** If the JD says "flow cytometry," write "flow cytometry" (not just "FACS"). Match the exact string, truthfully.

## Form
12. **One page.** Cut the least-relevant material until it fits. Relevance > completeness.
13. **Selectable text, standard fonts** (Helvetica/Inter/Arial/Calibri). No exotic fonts.
14. **Use a real em-dash `—` (U+2014)** in section titles, not the `&mdash;` HTML entity (it can break when uppercased by some renderers).
15. **Contact line = one line:** City, Country · email · phone · (optional) Scholar/ORCID. Keep links minimal.
